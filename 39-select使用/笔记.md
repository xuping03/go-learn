# 39. select 的使用

## 基本语法

`select` 监听多个 channel，哪个 case 能配对就执行哪个，都不能配对则阻塞：

```go
select {
case num := <-ch:
    fmt.Println("收到:", num)
case <-quit:
    return
}
```

## 实现超时

利用 `time.After` 在 select 中实现超时控制：

```go
select {
case num := <-ch:
    fmt.Println("num=", num)
case <-time.After(3 * time.Second):
    fmt.Println("超时")
    quit <- true
}
```

- 若 3 秒内 `ch` 没有数据，超时分支触发
- 常用于防止 goroutine 永久阻塞

## 综合示例：斐波那契数列

```go
func fibonacci(ch chan<- int, quit <-chan bool) {
    x, y := 1, 1
    for {
        select {
        case ch <- x:      // 有接收方时写入
            x, y = y, x+y
        case <-quit:       // 收到退出信号时结束
            return
        }
    }
}
```

主 goroutine 读取，读完后发退出信号：

```go
go func() {
    for i := 0; i < 8; i++ {
        fmt.Println(<-ch)
    }
    quit <- true
}()
fibonacci(ch, quit)
```

## 关键点

- `select` 只执行一个 case，若多个同时就绪则**随机选一个**
- 没有就绪的 case 时阻塞，有 `default` 分支则不阻塞直接走 default
- `time.After` 每次调用都创建新 timer，在 select 内使用不会累积
