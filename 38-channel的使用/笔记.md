# 38. channel 的使用

## 创建 channel

```go
ch := make(chan int)    // 无缓冲 channel
ch := make(chan int, 3) // 有缓冲 channel，容量为 3
```

- `len(ch)`：当前缓冲中的数据个数
- `cap(ch)`：缓冲容量

## 无缓冲 vs 有缓冲

| | 无缓冲 | 有缓冲 |
|---|---|---|
| 发送方 | 阻塞直到接收方就绪 | 缓冲未满时不阻塞 |
| 接收方 | 阻塞直到发送方就绪 | 缓冲为空时阻塞 |

## 同步用法

利用 channel 的阻塞特性，保证 goroutine 执行顺序：

```go
ch := make(chan string)
go func() {
    // 做一些工作...
    ch <- "工作完毕" // 完成后通知主 goroutine
}()
str := <-ch // 主 goroutine 阻塞等待
fmt.Println(str)
```

## 关闭 channel

```go
close(ch) // 关闭后不能再写，但可以继续读剩余数据

// 方式一：判断是否关闭
if num, ok := <-ch; ok {
    fmt.Println(num)
}

// 方式二：range 自动检测关闭
for num := range ch {
    fmt.Println(num)
}
```

- 向已关闭的 channel 写入会 panic
- 从已关闭且为空的 channel 读取，返回零值

## 单向 channel

```go
var writeCh chan<- int = ch  // 只写
var readCh  <-chan int = ch  // 只读
```

函数参数中使用单向 channel 限制读写权限：

```go
func producer(out chan<- int) { out <- 1 }
func consumer(in <-chan int)  { num := <-in; _ = num }
```

## timer 定时器

```go
timer := time.NewTimer(2 * time.Second)
t := <-timer.C // 阻塞 2 秒后收到当前时间

timer.Stop()                  // 停止定时器（channel 不会再收到值）
ok := timer.Reset(1 * time.Second) // 重置为新的时长
```

`time.After` 是 timer 的简写，返回一个 channel：

```go
i := <-time.After(2 * time.Second) // 阻塞 2 秒
```

## ticker 周期定时器

```go
ticker := time.NewTicker(time.Second) // 每隔 1 秒触发一次
for i := 0; i < 5; i++ {
    <-ticker.C
    fmt.Println(i)
}
ticker.Stop()
```

- `timer`：只触发一次
- `ticker`：周期性重复触发
